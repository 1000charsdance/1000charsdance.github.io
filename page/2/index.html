防晒霜

1、用量：一硬币、2mg/平方厘米，脸部面积一般为360平方厘米，即涂0.72g，加上脖子0.24g，越1次1g。75g的大宝防晒霜，用75天。

2、时间：全年均要涂，无论晴雨。

3、补涂：每过3-5小时补涂，如果主要是室内活动也可以不补涂。如果没有化妆，用补水喷雾和纸巾清理脸上的灰尘、汗水、油脂，再涂防晒霜，上少许散粉吸油就可以了。如果化了妆，怕妆花了，不要卸妆，能不卸妆就不卸妆，可以用带防晒系数的粉饼补妆。

4、卸妆：防水、防汗、防水防汗标识需要用卸妆水卸妆，像大宝清爽透气防晒霜用洗面奶即可清洗。

5、涂抹方法：不要来回用力按摩，防晒霜含大分子，容易堵住毛孔，用按压方式即可。顺一个方向抹开。

6、油腻怎么办？

如果只涂防晒，不化妆，如果感觉油腻，也是可以定妆的。用粉扑粘少许的散粉，用十字交叉法揉搓粉扑，让散粉均匀铺在粉扑上，然后用粉扑轻而薄地打上脸部，起到控油的效果，能吸收脸部的油脂，减少脸部油光和油腻感，同时也能一定程度令防晒的效果更持久。反之，如果在涂完防晒霜后，脸部没有感到油腻，则就不需要定妆。

此外，在涂防晒霜后用散粉定妆，还能起到轻微的遮瑕的效果，但要注意不能拍过多的散粉，否则会使脸部出现发白，变得不自然。

八卦的人受欢迎

路人甲和路人乙讲路人丙的八卦，那么路人甲和路人乙能很快地拉近关系，会八卦的人很受欢迎，经过多年的社会选择，八卦成了刻进DNA的能力。

八卦与故事

八卦其实就是对人的关心，关心别人身上发生的故事，所以会八卦的我们难以记住生硬的道理，而对与人相关的故事比较敏感。

故事记忆法适用于研究人与人之间的学科，比如研究人与人之间最低的道德标准的法学、研究人怎么花钱和赚钱的经济学、研究人与人之间怎么合作的管理学等等。

重构记忆法

问题引入

请回答以下三个问题。

1、初一数学教材第二章有没有出现=号？

2、初一英语第一章有没有出现the这个单词？

3、2008年8月8日，你有没有呼吸过一次？

看到前两个问题，你可能会有点蒙，当看到第三个问题时，会想：哦，2008年8月8日是北京奥运会开幕式，我在。。。诶？我不需要直到我在干什么呀，我不呼吸就挂了，当然有呼吸啊。

前两个问题也是如此，数学里每一个等式都有=号，，而英语文章里动不动就有定冠词the。

我们不会也不需要去记忆这种事情，但是根据推理，不难得出这些问题的答案，这种方法就是重构记忆法。

方法描述

根据一个规则和有限个锚点，从而推导出大量甚至无穷知识的方法，就是重构记忆法。

比如死记硬背类似于背圆周率，3.1426535……，而使用重构记忆法类似于背斐波那契数列1，1，2，3，5，8，13……，我们只需要记住一条规则：每个数字都等于前面两个数字之和，以及两个锚点，前两个数字是1，我们就记住了无穷个数字。

重构记忆法既可以记忆理工科知识，也可以记忆文科类知识。

举例

理工类：计算98*45。

记住竖式乘法规则和乘法口诀表，我们就可以把结果计算出来，而不需要去记98*45等于多少。

文科类：驾考科目一，在高速公路上低于限速行驶扣3分，停车扣6分，掉头倒车扣12分。这背后的逻辑是你的行为越危险，收到惩罚的力度就越大，危险性上低于限速行驶<停车<倒车。我只要记住这点，再记住低于限速行驶扣3分，就可以推出其它两个分别扣6分和12分了。

睡眠

对应5R中的Rest

睡前

1、睡前主动回忆今天发生的事情，几点起，第一节课先讲了什么，后讲了什么……，

2、把要记忆的事情放到睡前做，比如背单词。

睡醒

刚睡醒时记忆力、意志力、精神状态都是最好的，不要浪费在不重要但花费精力的事情上，比如各种决策：闹钟响了要不要再睡10分钟，选什么上衣搭配这条裤子，包里要带什么东西，这些要前一天准备好。而洗漱、吃早餐甚至运动这些不需要耗费精力的习惯动作可以做。把一天中最难的事情放到早上来做，不要浪费精力。

午觉

作用

1、将早上记忆的内容转成长期记忆，清理大脑内存，提高下午反应力和精神状态。

2、如果前一天晚上睡眠不足，午觉可以适当弥补。

睡眠时长

时间不重要，阶段才重要。

睡眠周期分四个阶段：打盹、浅层睡眠、深度睡眠、快速眼动睡眠。

只有深度睡眠下醒来会感觉整个人非常蒙，所以如果午觉起来感觉很困，要么减少睡眠时间到不发生深度睡眠，要么增加睡眠时间到发生快速眼动睡眠或下一个睡眠周期，具体时间因人而异。

如果早上学习记忆任务很重，就延长午睡时间，能提高学习效果，改善下午和完善的状态。

如果前天晚上睡眠不足，午觉提前并延长时间。

提炼

对应5R中的Reduce

备考：

不要从头到尾地一章章地背。

1、背诵简单且常考。

2、放弃已经熟记的知识。

3、提炼关键词句。

推广非备考：

1、分多轮记忆知识，按重要性降序记忆，第一轮仅需要先初步形成知识网络的框架，而不用搭建完整的知识网络，难度小很多，后续经过每一轮记忆逐步完善这个网络。

2、每一轮放弃熟记，提炼关键词句，逐渐背困难且少考的内容，直到性价比不高。

重复

对应5R中的Repeat

重复

学习分三个阶段：负反馈、正反馈、无反馈

负反馈

专注于困难的计划和远大的目标，未开始或刚开始被计划劝退，坚持了两天没效果就放弃。

原因：不注重当下学习的动作。

方法：你看视频学习后有没有动手去实践？看完一章书后有没有总结本章内容？做错的题目看了答案后有没有再做一遍？记忆过的知识有没有进行主动回忆？

正反馈

坚持注重学习的动作后，成绩有所上升，但又会遇到瓶颈，于是不断地对已经熟记的知识或笔记学习背诵，成绩还是没有提升。

原因：没有跳出舒适圈。

方法：制定知识的分级系统，分为困难模糊、熟悉、完全记住三级，精力放在前两级上。根据自身学习掌握情况，动态更新这个分级系统。

无反馈

对于完美计划、困难目标、学习没见效视而不见，从一砖一瓦做起，专注于每一个学习的动作。

成绩提高不骄傲，成绩下滑不气馁，遇到瓶颈不沮丧。

间隔重复

比喻

大脑新的记忆会抑制旧的记忆，你每过一段时间去提取旧的记忆，大脑需要努力去恢复记忆，几次后就烦了，干脆放那里，你要就自己拿，记忆就牢固了。

举例

背单词

500个单词10天背完怎么背？每天50个？

应该按记忆强度来背，每一轮都背500个单词，第一轮可能花5天，记忆强度为20%。

开启第二轮记忆，发现有的单词完全记住，有的记住部分拼写，有的记住部分中文意思，有的什么都没记住。对于完全记住的单词可以做一个标记，有标记的或者说有3个标记再下轮就不背。第二轮记忆强度可能花3天，记忆强度为40%。

记住的越来越多，要记的内容越来越少，每一轮的速度加快，第三轮花2天，记忆强度60%，第四轮花1天，记忆强度80%，第五轮就是个把小时背一下重点词汇。

背大段文字

第一轮：记住有5个关键词。

第二轮：5个关键词记得3个，另外2个是？

第三轮：5个关键词都记住，细节是？

第四轮：5个关键词的子关键词有几个？

第五轮：……

集中重复

间隔重复~记忆强度/记忆广度/记忆的量

集中重复~记忆深度

过度重复~记忆速度

原理

大脑的工作的脑容量为7个单位，如果使用碎片化时间进行学习，周围环境的内容会占用工作脑容量，把刚学习的内容挤出去，再学习下一个知识，和上一个知识就形不成联系，无法加入到知识网络中。

方法

我们应该花集中大块的时间进行学习，大脑中都是学习的内容，我们可以深入地寻找知识之间的联系、区别，去比对区别、猜测联系、进行实验、分析反馈、重构规律，这样知识之间就形成了紧密深度的联系，也就是深度记忆。

举例

学霸看书

很多人看书就一直往后翻，学霸则经常来回地翻动课本，他是在不断地比对知识，诶这题好像刚才做过，真的做过吗？有什么区别吗？有共同的规律吗？经过反复地研究，最终重构出一个新的规则，把这些知识串联起来。

专题学习

习题集把一个专题的题目放在一起，刷上一个晚上后，你发现这个专题的题目之间有共同的解题思路，几个题型之间既有共同点，有有各自的特点，相互之间有所区别。

攻克错题

选择题两个选项模棱两可？选项中没有自己预想的，是题目出错了？这道题目好像刷过，下意识地选了答案，结果错了，归结于自己粗心大意。

原因其实不是粗心大意，而是没有足够的刺激辨别。对于关联的知识点，找到它们的联系或规律，这叫做刺激泛化。而如果对于一个规律内的知识点无法明确地区分，这就是过度泛化了，需要刺激辨别。

对于关联的知识点，必须非常肯定、确定它们的区别。比如经常搞不懂dessert和desert、quiet和quite等等的区别，每次遇到就纠结，诶呀好像是这个，这就说明刺激辨别不足，需要把所有相近的单词放一块，找个时间集中起来进行比对攻克。

过度重复

思维密度：对一个知识点，每提出并回答一个问题，思维密度加1。比如看一个单词，思维密度是0。遮起来默写单词，思维密度是1，再写出中文意思，思维密度是2，写出近义词，思维密度是3，写出反义词，思维密度是4，写出固定搭配，思维密度是5。

思维密度越高，记忆的效果越好，记忆的速度越快。

C++20 的正确处置办法就是

  1. （老生常谈）正确性很重要，编码转换省不得。、平台 API、boost.locale、ICU 都可以帮你。
  2. 内部尽量用 Unicode。Unicode 编码应作为中间格式。表达文本的字符串的字面量应当用 Unicode 版本，且源文件应以 Unicode 编码保存，否则可能因编辑器、编译器、其它系统（例如拷源码到别的机器上）的编码设置不一致导致乱码。
  3. （现实考量）Linux 上的 wcin/wcout 破烂不堪，从来没有靠谱过。Windows API 以 USC2（美其名曰 UTF-16） 为本，用 cin/cout 虽然有性能损失，但为了跨平台迁就 Linux，还是选用 cin/cout 较好。（坚实可靠 Windows 支持 cin/cout 和 wcin/wcout 混用，但其他系统不一定。）
  4. locale 可以设置为 Unicode 变体。此时 std::cout 的 char 的编码是 UTF-8。而 std::wcout 的 char 的编码是 UTF-8，wchar_t 编码是 UTF-16（Windows）或 UTF-32（多数 Unix-like）。这样，char8_t & 在输入/输出能 reinterpret_cast 成 char &，而 char16_t & 或 char32_t & 中的一种（视平台而定）可以 reinterpret_cast 成 wchar_t &。string/u8string/u16string/u32string 同理处置。这样能解决大多数需求。
  5. Language lawyers 先别激动。更正确的方法当然是用 std::codecvt 等编码转换功能来转换。
  6. string 间的转换和重载好的运算符，标准库虽未提供，但你可以自行添加。未来可能标准库可能会添加一部分。

std::codecvt 的取得：

  1. std::codecvt<char16_t, char8_t, std::mbstate_t>、std::codecvt<char32_t, char8_t, std::mbstate_t> 和 std::codecvt<wchar_t, char, std::mbstate_t> 直接由 提供，可以放心使用。std::codecvt<char16_t, char, std::mbstate_t> 其实等同于 std::codecvt<char16_t, char8_t, std::mbstate_t>，并不是真的转换到特定编码的 char，已被 C++20 废止。其他 std::codecvt_u??? 也是同理，均已废止。（std::codecvt_byname 各模板实例的情况与 std::codecvt 的平行，不再赘述。）
  2. 其他 std::codecvt 通过 use_facet<codecvt<TChar1, TChar2, mbstate_t>>(locale) 取得，标准库不保证各种 char 之间的转换都已实现。你可用使用 boost.locale 补充。
