由点到平面的距离引发的思考

发表于 2021-04-17 更新于 2022-10-22

因为要渲染数学公式所以加载速度有点慢，无法显示数学公式可以多刷新几遍

发现问题

从一道高等数学的习题说起
$$
求旋转抛物面z=x^2+y^2与平面x+y-z=1之间的最短距离.
$$

开始时的胡思乱想

开始时我企图通过模仿求抛物线到直线的距离，用几何（作图）的方式解决问题 ，先求出导数相等的点。

平面的法向量为$\vec{n}=(1, 1, -1)$
$$
\begin{cases}z_x=2x=1\z_y=2y=1\end{cases}
$$

解得
$$
\begin{cases}x=\frac {1}{2}\y=\frac {1}{2}\z=\frac {1}{2}\end{cases}
$$
虽然点的坐标是正确的，但是我并不相信这种乱来的方法，于是我查看了答案。

标准答案

设M(x, y, z)为抛物面上任一点，则距离的平方 ==$d^2=\frac{(x+y-z-1)^2}{3}$== ，求其在条件 $z=x^2+y^2$ 下的最值。

构造拉格朗日函数$F(x,y,z)=\frac{(x+y-z-1)^2}{3}+\lambda (z-x^2-y^2)$

求驻点得 $x=y=z=\frac{1}{2}$ 此处省略解方程组（不想敲）

则 $d=\frac{\sqrt3}{6}$

emm…原来如此。（看答案真快乐

我的答案

设 M(x, y, $x^2+y^2$)​，则 $d=\frac{|x+y-x^2-y^2-1|}{\sqrt3}=\frac{|(x-\frac{1}{2})^2+(y-\frac{1}{2})^2+\frac{1}{2}|}{\sqrt3}\ge\frac{\sqrt3}{6}$

好像异常简单，只用了一个点到平面的距离公式，之前我怎么没想到呢？哦~，原来我不会啊，那没事了。

等等，这个公式为什么会是这样子呢？

思考问题

两个形式上高度统一的公式

点到直线的距离公式

点(x, y)，直线方程：$Ax+By+C=0$，点到直线的距离公式：

==$d=\frac{|Ax+By+C|}{\sqrt{A^2+B^2}}$==

点到平面的距离公式

点(x, y, z)，平面方程：$Ax+By+Cz+D=0$，点到平面的距离公式：

==$d=\frac{|Ax+By+Cz+D|}{\sqrt{A^2+B^2+C^2}}$==

从几何的角度解释

这两个公式用几何的方式解释都形象且简单，教科书以及老师都是这么解释的，我不再赘述。

更高维的距离公式

点(x, y, z, w), 到方程：Ax+By+Cz+Dw+E=0 的距离公式为

==$d=\frac{|Ax+By+Cz+Dw+E|}{\sqrt{A^2+B^2+C^2+D^2}}$==

啥？你问我Ax+By+Cz+Dw+E=0代表什么，不好意思，我不知道，这个公式也没有人教我，是我按照规律编出来的。

物理上第四个维度是时间，我这里指的是纯数学的高维，我有自己的一些思考，所以写下这篇博客。

我无法想象和理解高于四维的空间（四维空间和四维时空听一些大佬讲还是能够窥探到四维的一些奥妙的，比如B站的up主 眼见为识 https://space.bilibili.com/430948380?spm_id_from=333.788.b_765f7570696e666f.1），所以我想我现在是无论如何也无法用几何的方式来解释上述的四维距离公式的，更何况在数学上维度可以是无限的。

那么只能转变一下角度，用代数解释各维度的距离公式了。

从几何到代数的思维转变

距离的重新定义

距离是几何里的概念，代数是没有距离的概念的，就容我

放飞思想吧。

系数的奥妙

==$d=\frac{|Ax+By+C|}{\sqrt{A^2+B^2}}$==

==$d=\frac{|Ax+By+Cz+D|}{\sqrt{A^2+B^2+C^2}}$==

这两个公式的分母都是方程系数的平方和开根号，毫无疑问这些系数是一个突破口（貌似也只有这些系数可以研究）。

依次提取方程的系数组成系数向量$\vec{n}=(A, B<, C>)$,

方程的系数向量与满足该方程的点的集合（在几何上分别表现为直线和平面）是==”垂直“==的。即某点按系数向量的方向进行移动直到满足方程，所经过的路径是最短的，比按其它方向位移到满足方程要短。

==由此得到距离的另外一个定义：某点按照方程的系数向量移动至方程解集的线段长度。==

以点到直线的距离为例

直线方程：Ax+By+C=0

点$(x_0,y_0)$

如果$Ax_0+By_0+C=0$，距离d=0。

如果$Ax_0+By_0+C\ne0$，向(A, B)移动 t 个单位，使得

$A(x_0+At)+B(y_0+Bt)+C=0\tag1$

则垂点的坐标为$(x_0+At,y_0+Bt)$,距离$d=\sqrt{(At)^2+(Bt)^2}=|t|\sqrt{A^2+B^2}\tag2$

由(1)解得$t=\frac{Ax_0+By_0+C}{A^2+B^2}$，代入(2)，

$d=\frac{|Ax_0+By_0+C|}{\sqrt{A^2+B^2}}$

大功告成！

用类比的方法不难得出点到平面的距离公式，而且理论上适用于n维(元)==一次==方程。

这样，我们在没有借助几何的图像，以代数的角度解释了距离公式。

也许有人会说：搞笑，距离本来就是几何上的概念，脱离了几何，有什么现实意义吗？

也许有人会说：搞笑，到头来你不过是在原地踏步，没有实质进展。

也许有人会说：胡说八道！

而这个变换显然是与方程系数密切相关的。

常量成员

常量成员对于每个对象来说是常量，应该在定义对象时进行初始化，但是不同对象之间该成员的值可以不同。

静态成员

可以通过作用域运算符 ::进行操作

没有 this 指针，所以静态成员函数无法处理非静态成员

友元

友元可以为一般函数，其他类的成员函数和其他类。

  1. 没有传递性：B是A的友元，C是B的友元，不能说C是A的友元。
  2. 没有对称性：B是A的友元，A不一定是B的友元。
  3. 不是索取的而是授予的。

友元的声明最好写在类内最上方。

过程

1、环境搭建

Hexo 基于 Node.js，搭建过程中还需要使用 npm（Node.js 已带） 和 git，因此先搭建本地操作环境，安装 Node.js 和 Git。

安装完成后，Win+R 输入 cmd 并打开，依次输入 node -v、npm -v 和 git --version检验是否安装成功。

2、连接github

注册github账户

打开git bash，设置用户名和邮箱

创建 SSH 密匙：

输入 ssh-keygen -t rsa -C "GitHub 邮箱"，然后一路回车。

添加密匙：

进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

验证连接：

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。

显示 “Hi xxx! You’ve successfully……” 即连接成功。

GitHub 主页右上角加号 -> New repository：

填好后点击 Create repository 创建。
