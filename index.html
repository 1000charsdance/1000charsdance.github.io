博客标签怎么打

本篇讲讲我觉得博客标签该怎么打。

标签原则

原则1：标签的数量不是固定的，一篇博客可以没有标签，不需要强行打上不合适的标签。

原则2：标签不宜过多，一篇博客最好不要打超过3个标签。

原则3：不要把标签当作分类。分类是大类别包含小类别，而标签间不需要存在包含关系。

原则4：同种标签只打1个，选择最合适的那个标签。

标签制定

形式标签

第一种标签说明形式，共制定7个：

随笔杂谈（随心畅谈，不拘一格）

疑难追思（思考钻研）（重点：提出问题）（ps：与经验方法的区别是说只需要清楚问题的重难点和意义，可以留待日后解决）

经验方法（拿来主义、最为现实）（重点：经验的适用场景和应用方法）（ps：项目实践的经验部分如果单独抽离出来是经验方法的其中一种）

项目实践（经验来源、进步基石）（重点：详实过程）

理论学习（长远投资、照明铺路）（重点：系统的概念和清晰的体系）

软件使用（高效工作）

资源分享（分享主义）

内容标签

第二个标签说明内容，不要对内容分得太小。内容标签可以慢慢添加。

文学

C++面向对象高级开发

C++多态

多态是面向对象编程的三大特性之一。

多态使用父类的指针或引用调用子类对象的函数。

这个调用是动态绑定的。

函数重写和动态绑定是C++实现多态的主要方式，在父类中定义虚函数，在子类中重写虚函数，使用父类指针或引用调用子类对象的虚函数，实现动态绑定。

函数重载实现了根据参数列表调用不同的函数，但这个过程是在编译阶段完成的，算是伪多态。

构造函数、析构函数与虚函数

构造函数不能为虚函数，析构函数可以且建议定义为虚函数。

构造函数的作用是构造一个类的对象，它的发生时间是创建完对象之前，此时对象未创建成功，无法判断属于父类还是子类。而虚函数作用是使用父类指针或引用调用子类函数，如果把构造函数定义为虚函数，在构造对象时无法判断到底调用哪个类的构造函数，存在矛盾。

计算机网络

Q：TCP/IP模型有几层？

A：TCP/IP模型分四层，依次为应用层（HTTP、DNS、SMTP）、传输层（TCP）、网络层（IP）和网络接口层（以太网、MAC、数据帧）。

Q：TCP三次握手是什么？

A：TCP是可靠的传输协议，第一次握手为浏览器向Web服务器发送连接请求，第二次握手为Web服务器向浏览器发送接受信息，第三次握手为浏览器确认Web的接受信息，正式建立连接。三次握手的作用是确保浏览器和Web服务器都能正常发送和接受数据。

Q：Socket是什么？

A：Socket是介于应用层和传输层的抽象层，用于不同主机之间的网络通信，应用层通过Socket确定如何选用协议栈（TCP/UDP、IPV4/IPV6）。

Q：Socket函数是什么？

A：函数原型为int Socket(int domain, int type, int protocal)，domain表示协议族，包括IPV4、IPV6和本机，type表示数据类型，包括字节流（TCP）、数据报（UDP）和原始套接字，protocal弃用。

操作系统

发表于 2024-04-18 更新于 2024-05-28
  • 操作系统的四大功能：进程控制，内存管理，设备管理，文件管理
  • 进程的定义:并发程序的执行，进程的同步与互斥

进程的定义：进程是程序执行的一个实例。程序是可执行的二进制文件，运行程序时二进制文件从硬盘读取到内存中，二进制指令依次读入CPU并执行。

并发指一定时间间隔内一起发生。进程通过虚拟地址访问内存，不同进程中相同的虚拟地址对应不同内存。为避免进程访问其它进程的内存，一个CPU核心同时只能有一个进程。进程运行一定时间后，操作系统会切换其它进程到CPU运行，宏观上这些进程是并发的。并行则是实际上同时进行，线程是并行的，因为线程共用内存，不存在地址映射错误。

Q：进程的同步和互斥解决什么问题？

A：进程的同步和互斥属于进程的通信问题，解决进程的运行时机问题。

Q：进程的同步与互斥是什么？

A：进程同步指进程按照制定的顺序运行，比如总是运行进程A后运行进程B，总是听课后做笔记。进程互斥指运行的进程阻碍其它进程开始运行，比如进程A运行时进程B不能运行。进程的同步和互斥可以通过共享内存和信号量机制实现。

问：进程同步与互斥如何实现？

答：可以通过锁和信号量机制实现。

问：锁如何实现线程同步与互斥？

答：锁分为忙等待锁和无等待锁。

忙等待锁为使用While循环重复执行空语句，直到获得锁。忙等待锁依赖抢占式调度算法，否则会造成死循环。

问：信号量机制如何实现进程的同步与互斥？

答：信号量通常表示资源的数量，为整型数据。通过原子的P、V操作改变信号量的值，信号量小于0表示资源紧缺，令该进程进入等待队列，并通过操作系统进行进程调度。

P操作减少信号量并阻塞进程，V操作增加信号量并唤醒进程。

  • 进程的状态：创建，终止，挂起，激活，阻塞，唤醒，以及相互转换关系

  • 进程同步问题：生产者消费者模型，信号量机制

问：进程的同步问题有哪些？

答：进程的同步问题有生产者-消费者问题、哲学家问题、读者-写者问题，可使用信号量机制解决。

生产者-消费者问题用信号量表示产品的数量a和空槽的数量b，a不小于0时可以取走产品，b不小于0时可以生产产品。

  • 进程与线程的区别和联系

1、地位不同：进程是操作系统分配资源的基本单位，线程是CPU执行和调度的基本单位。、

2、耗时不同：进程的创建和调度需要消耗较多的时间，线程的创建和调度用时较短。

3、拥有资源不同：每个进程拥有独立的地址空间和资源。多个线程共享进程的大部分资源。

4、并发与并行：不同进程是并发的，进程间通过IPC机制进行通信。进程的多个线程是并行的。

5、崩溃后果：一个进程崩溃不会影响其它进程的运行，一个线程崩溃可能导致其它线程甚至进程的崩溃。

  • 线程池，线程的创建，同步，安全问题
  • 存储器管理：只读存储器，随机存储器，相对地址和绝对地址
  • 速度问题：CPU>寄存器>缓存>主存>磁盘缓存>硬盘
  • 程序的装入和链接（动态装入，静态装入，重定位）
  • 内存空间划分算法：首次适应；最佳适应；最坏适应
  • 段式存储和页式存储，以及段页式存储的优缺点，原理
  • 堆和栈的区别

1、内存分配：堆是动态分配内存的，大小可变，栈是静态分配内存，大小固定。

2、访问方式：堆通过指针访问，可以访问任意元素。栈固定访问栈顶的元素。

3、存储对象：堆保存对象的实例，栈保存函数调用和局部变量。

4、资源所属：堆是所有线程共有的，栈是每个线程都独立拥有。

5、生命周期：堆内存可以存在很久，自己手动回收内存。栈内存跟局部变量的生命周期相关，离开作用域就会被操作系统自动回收。
